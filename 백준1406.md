## 해결 포인트
**리스트보다는 스택이나 큐가 훨씬 빠르다**
- 리스트에서 중간에 값을 삭제하거나 추가하려 하면 나머지 값들도 다 이동시켜야 해서 오래 걸린다
- 리스트의 중간값을 다루는 것이라면, 리스트를 쪼개서 두개의 스택이나 큐로 생각할 수 있다.
- 데이터가 많을수록 input보다는 readline이 시간상 효율적이다

## 내 코드
```python
from collections import deque

import sys

s1 = deque(sys.stdin.readline().strip()) # 왼쪽 배열
s2 = deque() # 오른쪽 배열

n = int(sys.stdin.readline())

for _ in range(n):
    c = sys.stdin.readline().strip()

    if c == 'L':
        if len(s1) != 0:
            removed = s1.pop()
            s2.appendleft(removed)
    elif c == 'D':
        if len(s2) != 0:
            removed = s2.popleft()
            s1.append(removed)
    elif c == 'B':
        if len(s1) != 0:
            s1.pop()
    else:
        to_add = c.split()[1]
        s1.append(to_add)

s1.extend(s2)
print("".join(s1))
```
